# Тест-кейсы для API объявлений

## Позитивные сценарии

| ID | Название | Шаги | Ожидаемый результат |
|----|----------|------|---------------------|
| TC-P-01 | Создание объявления с валидными полями | POST /api/1/item с {sellerID, name, price} | 200 OK, в ответе есть id, sellerId, name, price |
| TC-P-02 | Получение объявления по id | Создать объявление, затем GET /api/1/item/{id} | 200 OK, ответ - массив с одним элементом, поля совпадают |
| TC-P-03 | Получение всех объявлений продавца | Создать 2 объявления с одним sellerID, затем GET /api/1/{sellerID}/item | 200 OK, список содержит оба объявления |
| TC-P-04 | Получение статистики по существующему id | Создать объявление, затем GET /api/1/statistic/{id} | 200 OK, массив с объектом {likes, viewCount, contacts} |
| TC-P-05 | Удаление объявления | Создать, затем DELETE /api/2/item/{id} | 200 OK, после удаления GET возвращает 404 |

## Негативные сценарии

| ID | Название | Шаги | Ожидаемый результат |
|----|----------|------|---------------------|
| TC-N-01 | Создание без sellerID | POST без поля sellerID | 400 Bad Request |
| TC-N-02 | Создание с пустым name | POST с name="" | 400 |
| TC-N-03 | Создание с price = 0 | POST с price=0 | 400 (или 422) |
| TC-N-04 | Создание с price = -100 | price=-100 | 400 |
| TC-N-05 | Создание с price = строка | price="abc" | 400 |
| TC-N-06 | Создание с sellerID вне диапазона (например, 1) | sellerID=1 | 400 |
| TC-N-07 | Получение несуществующего id | GET /api/1/item/0000 | 404 |
| TC-N-08 | Получение объявлений с несуществующим sellerID | GET /api/1/999999999/item | 200 с пустым массивом |
| TC-N-09 | Статистика по несуществующему id | GET /api/1/statistic/invalid | 404 |
| TC-N-10 | Удаление несуществующего объявления | DELETE /api/2/item/12345 | 404 |

## Корнер-кейсы

| ID | Название | Описание | Ожидаемый результат |
|----|----------|----------|---------------------|
| TC-C-01 | Длинное имя (1000 символов) | name = "A"*1000 | 200 или 400 (должен быть лимит) |
| TC-C-02 | Максимальная цена (10^9) | price = 1000000000 | 200 |
| TC-C-03 | Дробная цена (5000.50) | price = 5000.50 | 400 (если только целые) |
| TC-C-04 | Идемпотентность POST | Два одинаковых запроса подряд | Разные id |
| TC-C-05 | Идемпотентность GET | Повторные GET /api/1/item/{id} | Одинаковый результат |
| TC-C-06 | Лишние поля в теле POST | Добавить поле "extra" | 200 (игнорируется) или 400 |
| TC-C-07 | XSS в name | name="<script>alert(1)</script>" | 200, при получении значение экранировано |
| TC-C-08 | Границы sellerID: 111111 и 999999 | Минимальное и максимальное | 200 |
| TC-C-09 | Статистика после просмотра | GET item → GET statistic | viewCount увеличивается |

## Функциональные цепочки (E2E)

| ID | Название | Шаги | Ожидаемый результат |
|----|----------|------|---------------------|
| TC-E2E-01 | Создать -> Получить -> Удалить -> Получить (404) | 4 шага | Все успешны, последний GET = 404 |
| TC-E2E-02 | Создать -> Статистика -> Просмотр -> Статистика обновлена | 4 шага | viewCount увеличился |

## Нефункциональные проверки

| ID | Название | Описание | Ожидаемый результат |
|----|----------|----------|---------------------|
| TC-NF-01 | Время ответа POST | Среднее из 10 запросов | < 500 мс |
| TC-NF-02 | Время ответа GET /{sellerID}/item | При наличии 10+ объявлений | < 300 мс |
| TC-NF-03 | SQL injection в id | GET /api/1/item/1' OR '1'='1 | 400 или 404, не 500 |
