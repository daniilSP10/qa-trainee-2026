# Найденные дефекты (API)

## BUG-1: GET /api/1/item/{id} возвращает массив, а не объект

**Серьёзность:** P2 (средний) – не соответствует REST-конвенции  
**Окружение:** https://qa-internship.avito.com, метод GET /api/1/item/{id}

**Шаги воспроизведения:**
1. Создать объявление через POST /api/1/item
2. Выполнить GET /api/1/item/{id}

**Фактический результат:** Ответ – массив с одним объектом: `[{...}]`  
**Ожидаемый результат:** Ответ – объект `{...}` (без обёртки массива)

## BUG-2: Статистика возвращает массив из двух одинаковых объектов (на /api/1/statistic/{id})

**Серьёзность:** P2 (средний) – избыточные данные  
**Шаги:**
1. Создать объявление
2. GET /api/1/statistic/{id}

## BUG-5: GET /api/1/{sellerID}/item для несуществующего sellerID возвращает чужие объявления вместо пустого списка или 404

**Серьёзность:** P1 (высокий) — утечка данных  
**Шаги:** GET /api/1/999999999/item  
**Фактический результат:** 200 OK, список из 42 объявлений других пользователей  
**Ожидаемый результат:** 200 OK с пустым списком [] или 404

## BUG-6: GET /api/1/item/nonexistent возвращает 400 вместо 404

**Серьёзность:** P2  
**Шаги:** GET /api/1/item/invalid_id  
**Фактический результат:** 400 Bad Request  
**Ожидаемый результат:** 404 Not Found
