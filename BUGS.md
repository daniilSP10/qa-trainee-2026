# Найденные дефекты (API)

## BUG-1: GET /api/1/item/{id} возвращает массив, а не объект

**Серьёзность:** P2 (средний) - не соответствует REST-конвенции  
**Окружение:** https://qa-internship.avito.com, метод GET /api/1/item/{id}

**Шаги воспроизведения:**
1. Создать объявление через POST /api/1/item
2. Выполнить GET /api/1/item/{id}

**Фактический результат:** Ответ - массив с одним объектом: `[{...}]`  
**Ожидаемый результат:** Ответ - объект `{...}` (без обёртки массива)

---

## BUG-2: Статистика возвращает массив из двух одинаковых объектов на /api/1/statistic/{id}

**Серьёзность:** P2 (средний) - избыточные данные  
**Окружение:** https://qa-internship.avito.com, GET /api/1/statistic/{id}

**Шаги воспроизведения:**
1. Создать объявление
2. GET /api/1/statistic/{id}

**Фактический результат:** 
```json
[
  {"likes": 0, "viewCount": 0, "contacts": 0},
  {"likes": 0, "viewCount": 0, "contacts": 0}
]
```
**Ожидаемый результат:** один объект или массив с одним элементом

## BUG-3: При создании объявления поле sellerID в запросе, а в ответе sellerId, разный регистр

**Серьёзность:** P3 (низкий) - несоответствие нейминга  
**Окружение:** https://qa-internship.avito.com, POST /api/1/item

**Шаги воспроизведения:**
1. Отправить POST /api/1/item с телом, содержащим поле `"sellerID"`
2. Посмотреть ответ

**Фактический результат:** В ответе поле называется `"sellerId"`  
**Ожидаемый результат:** Имя поля должно совпадать с запросом – `"sellerID"`

**Окружение:** https://qa-internship.avito.com, DELETE /api/2/item/abc

1. Выполнить DELETE /api/2/item/abc

**Фактический результат:** HTTP 200 OK  
**Ожидаемый результат:** HTTP 400 Bad Request

## BUG-8: Невозможно создать объявление - сервер отвергает все валидные запросы. Критический баг

**Серьёзность:** P0 (критический) — основной функционал не работает  
**Окружение:** https://qa-internship.avito.com, POST /api/1/item

**Шаги воспроизведения:**
1. Отправить POST /api/1/item с телом, соответствующим документации (поле `sellerID`, `name`, `price`, вложенный `statistics` с `likes`, `viewCount`, `contacts`)
2. Либо отправить те же поля на верхнем уровне

**Фактический результат:** 
- Для варианта с `statistics`: 
  ```json
  {"result": {"message": "поле likes обязательно", "messages": {}}, "status": "400"}
  ```
- Для варианта с полями на верхнем уровне:
```json
{"result": {"message": "", "messages": {}}, "status": "не передано тело объявления"}
```
**Ожидаемый результат:** HTTP 200 OK, объявление создаётся, в ответе приходит id.

**Дополнительно:** Проведены 6 различных комбинаций типов и имён полей, ни одна не привела к успеху. Сервис неработоспособен.
