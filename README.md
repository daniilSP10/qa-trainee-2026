## API тесты

Автотесты для API микросервиса объявлений. Проверяются эндпоинты:

- `POST /api/1/item` – создание объявления
- `GET /api/1/item/{id}` – получение объявления по id
- `GET /api/1/{sellerID}/item` – все объявления продавца
- `GET /api/1/statistic/{id}` – статистика (версия 1)
- `DELETE /api/2/item/{id}` – удаление (версия 2)

## Известные проблемы

Сервис `https://qa-internship.avito.com` содержит критические баги, из-за которых **невозможно создать объявление** (все запросы возвращают 400). Соответственно, тесты, зависящие от создания данных (позитивные тесты, E2E, статистика, удаление), падают. Это не ошибка тестов, а дефект тестируемого API. Детали в `BUGS.md`.

## Установка и запуск

1. **Клонировать репозиторий**

   ```bash
   git clone https://github.com/ваш-логин/название-репозитория.git
   cd название-репозитория
   ```

2. **Установить зависимости**

   ```bash
   pip install -r requirements.txt
   ```

3. **Запустить все тесты**

   ```bash
   pytest
   ```

   **Запуск с подробным выводом:**

   ```bash
   pytest -v
   ```

   **Запуск с генерацией отчёта Allure:**

   ```
   pytest --alluredir=allure-results
   allure serve allure-results
   ```

   **Запуск конкретного тестового файла:**

   ```
   pytest tests/test_create_item.py
   ```

   **Запуск конкретного теста:**

   ```
   pytest tests/test_create_item.py::TestCreateItem::test_create_valid_item
   ```

4. **Проверка стиля (flake8)**

   ```
   flake8 tests/ utils/
   ```

## Примечания

- Тесты используют случайные `sellerId` в диапазоне 111111–999999.
- Каждый тест создаёт свои данные и по возможности удаляет их после себя (через фикстуру `created_item`).
- Результаты тестов падают из-за багов в сервисе – это нормально, баги описаны в `BUGS.md`.
- Тесты независимы и воспроизводимы.

## Структура

- `TESTCASES.md` – детальные тест-кейсы (позитивные, негативные, корнер-кейсы, нефункциональные)
- `BUGS.md` – найденные дефекты с шагами воспроизведения
- `tests/` – автоматизированные тесты (pytest)
- `utils/` – вспомогательные модули (API клиент, helpers)
